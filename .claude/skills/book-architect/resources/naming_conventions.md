# Naming Conventions — fastapi-tutor

> Единый словарь терминов и правил именования для всех скилов и команд продукта `fastapi-tutor`.
> Источник истины. Все скилы (`project-brief`, `requirements-interview`, `book-architect`, `book-author`) и команды (`/tutor-*`) обязаны использовать эти термины без синонимов.

## 1. Артефакты конвейера

| Термин | Файл (по умолчанию) | Кто создаёт | Описание |
|--------|---------------------|-------------|----------|
| Файл №1 / project brief | `project_brief.md` | `/tutor-brief` | Описание будущего проекта и стека |
| Файл №2 / requirements | `requirements.md` | `/tutor-requirements` | Опросник по 4 категориям требований |
| План книги / outline | `book/outline.md` | `/tutor-plan` | Машиночитаемый план книги с чек-боксами |
| Книга / book | `book/` | `/tutor-write` | Дерево глав/секций/шагов |
| Отчёт валидации | (выводится в чат) | `/tutor-validate` | Список расхождений + вердикт PASS/FAIL |

## 2. Структура книги

```
book/
├── outline.md
├── chapter_1_{name}/
│   ├── section_1_{name}/
│   │   ├── {step_name}.md
│   │   └── {step_name}.md
│   └── section_2_{name}/
│       └── {step_name}.md
└── chapter_2_{name}/
    └── ...
```

| Уровень | Термин | Шаблон каталога/файла | Пример |
|---------|--------|------------------------|--------|
| 1 | Глава (chapter) | `chapter_<N>_<snake_name>/` | `chapter_1_starting_project/` |
| 2 | Секция (section) | `section_<M>_<snake_name>/` | `section_2_make_first_api/` |
| 3 | Шаг (step) | `<KK>_<snake_name>.md` | `01_create_first_api.md` |

## 3. Правила именования

1. `<N>` глав — целые числа, **сквозные без пропусков и дублей**. Нумерация начинается с `1`; допускается спец-глава `chapter_0_prerequisites` (подготовка окружения) — тогда отсчёт идёт с `0`. `<M>` секций — с `1`.
2. **Файлы-шаги ОБЯЗАТЕЛЬНО начинаются с 2-значного префикса порядка** `<KK>_` (`01_`, `02_`, …): сквозной, с `01`, без пропусков и дублей **внутри секции**. Префикс делает порядок шагов видимым прямо в списке файлов — не нужно открывать файл, чтобы понять, какой первый. Пример секции: `01_start_feature_branch.md`, `02_incomes_schema.md`, …, `10_commit_push_open_mr.md`.
3. `<name>` — `snake_case`, латиница, глагол+объект, отражает действие: `create_user_api`, `setup_docker`, `deploy_to_prod`.
4. Имена файлов-шагов уникальны в пределах своей секции.
5. **«Шаг N из M в секции»** в заголовке файла — НЕ пишется вручную: `N` = число из префикса `<KK>`, `M` = количество шагов в секции. Эти значения обязаны совпадать с фактическим расположением файла (проверяется валидатором).
6. Все ссылки между файлами книги — **относительные** от корня проекта (`book/...`). Абсолютные пути запрещены.
7. **Служебные шаги** (не привязаны к слою, допустимы в любой секции; префикс `<KK>_` им тоже обязателен): `how_we_work_git_flow`, `request_flow_overview`, `recap_and_state`, `start_feature_branch`, `commit_push_open_mr`, `setup_environment`.

## 4. Формат строки `outline.md` (контракт для `/tutor-write`)

Каждый шаг — одна строка:

```
- [ ] <относительный_путь_к_шагу>.md — <одна строка назначения>
```

- `[ ]` — шаг не написан; `[x]` — шаг написан и принят.
- Регэксп-контракт: `^- \[( |x)\] (\S+\.md) — (.+)$` (путь шага оканчивается на `<KK>_<name>.md`).
- Заголовки глав в outline: `## chapter_<N>_<name>`.
- Внутри секции шаги перечислены **по возрастанию префикса** `<KK>` — порядок в outline = порядок префиксов = порядок файлов в папке.
- Тире-разделитель назначения — символ `—` (em dash, U+2014).

## 5. Запрещено

- Менять термины на синонимы («раздел» вместо «секция», «урок» вместо «шаг»).
- Использовать пробелы или кириллицу в именах каталогов/файлов книги.
- Хардкодить технологический стек — он берётся из Файла №1.
