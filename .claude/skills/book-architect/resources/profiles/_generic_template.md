# Architecture Profile: _generic_template (заполняемый)

> Используется, когда курируемого профиля под выбранный стек нет (SSR-фреймворк Next/Nuxt, мобильное, CLI, гибрид и т.п.). `/tutor-stack` **заполняет** этот шаблон на основе Stack Knowledge Pack (исследование Context7/веба) и сохраняет результат как профиль проекта в `stack_decision.md` (раздел «Architecture Profile»). Заполняй ЧЕСТНО: если поле не выводится из исследования — пометь `[уточнить]`, не выдумывай.

## Обязательные инварианты (не зависят от стека)

Любой заполненный профиль ОБЯЗАН сохранить общие правила code-tutor:

1. Первая фича трека — **один слой = один шаг**, полный код + «Разбор кода».
2. Доменная глава открывается обзором потока (`mental_model_ref`).
3. Каждая глава завершается `recap_and_state` (полный `state_snapshot_file` + дерево).
4. Каждая фича-секция обрамляется `start_feature_branch` / `commit_push_open_mr`.
5. Каждый шаг заканчивается **проверяемым результатом**.
6. Если есть API/контракт — реализация сверяется с `book/contracts/<domain>.yaml`.
