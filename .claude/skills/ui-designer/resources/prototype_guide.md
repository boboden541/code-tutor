# Гайд: три прототипа-направления для выбора

> Прототип нужен, чтобы пользователь **увидел** будущий интерфейс и выбрал направление до написания книги. На выходе — **три системно разных направления** (см. `art_direction.md`), пользователь выбирает одно. Две поверхности: основная — Claude Design (DesignSync), fallback — Artifact. В любом случае `design_spec.md` (Файл №4) пишется по выбранному направлению.

## Что собираем — ТРИ направления
Для **каждого** из трёх направлений — самодостаточная HTML-страница (inline CSS на токенах этого направления; шрифтовая пара через `<link>` Google Fonts), показывающая:
1. **Шапку направления** — ярлык + одно-строчное позиционирование («на кого и какая эмоция»).
2. **Дизайн-систему** — карточки компонентов с **состояниями** (Button default/hover/active/focus/disabled/pending), палитра, **шрифтовая пара в деле**, триггеры доверия (рейтинг/бейдж/отзыв).
3. **Ключевые экраны** из матрицы покрытия (главные пользовательские сценарии — конкретный набор зависит от домена: напр. витрина/лендинг с продающими блоками, список/каталог, деталь сущности с CTA, форма создания, деталь с действиями по статусу, экран роли/дашборд) — с **реалистичными данными** (не Lorem Ipsum) и видимыми состояниями.

**Три направления различаются системно** (типографика/композиция/эмоция — ≥3 осей по `art_direction.md §A`), а не цветом. Каждое адаптивно (`<meta viewport>`, гибкие сетки, мобильная перестройка, нет горизонтального скролла). Стиль = токены выбранного направления (тот же визуальный язык, что пойдёт в код).

Файлы: `design/prototype/<domain>_dir1_<label>.html`, `_dir2_<label>.html`, `_dir3_<label>.html`.

> **Перед показом** прогони каждый прототип по запретам `art_direction.md §F`: нет симметричной 12-колоночной сетки как единственного приёма, плоских синих кнопок на белом, жёстких чёрных теней; у всех интерактивных элементов есть `:hover` + `cursor:pointer` + `:focus-visible`.

## Путь A (основной): Claude Design через DesignSync

DesignSync синхронизирует локальную папку HTML-превью с design-system проектом на claude.ai/design; пользователь смотрит карточки в Design System pane и утверждает.

1. Собери локально папку `design/prototype/` с HTML-файлами **по трём направлениям**. В первой строке каждого превью — маркер карточки с группой по направлению:
   `<!-- @dsCard group="Dir1·<label>" -->` (и `Dir2·…`, `Dir3·…`) — так пользователь сравнивает три направления отдельными группами в Design System pane.
2. `DesignSync method=list_projects` → есть ли writable design-system проект. Нет — `create_project name="<Domain> UI"`.
3. `DesignSync method=finalize_plan projectId=<id> writes=["**/*.html"] localDir=<.../design/prototype>` → получи `planId`.
4. `DesignSync method=write_files planId=<planId>` (файлы через `localPath`).
5. Дай пользователю ссылку на проект в claude.ai/design; попроси посмотреть и утвердить/дополнить.

> Первый вызов может запросить design-доступ к claude.ai-логину (или `/design-login`). Если доступа нет / инструмент недоступен в сессии (headless/cron) — переходи на путь B и отметь это в Файле №4.

## Путь B (fallback): Artifact

1. Собери **три** self-contained HTML-страницы (по направлению) — или ОДНУ с верхним переключателем направлений (таб «Направление 1/2/3»). Все CSS/JS inline; шрифты — через `<link>` Google Fonts (если внешние запросы недоступны — системная пара + пометка). Перед этим загрузи навык `artifact-design`/`frontend-design` для калибровки.
2. Запиши файл(ы) и вызови `Artifact` с путём, заголовком и favicon.
3. Дай пользователю ссылку(и) на опубликованные Artifact; попроси **выбрать одно** направление/дополнить.

## Чекпоинт выбора
Покажи **три направления** (ярлык + одно-строчное позиционирование + ссылка/карточки у каждого) + список экранов + матрицу покрытия. Дай рекомендацию (какое лучше подходит сути продукта и почему), но решает пользователь: «выбери направление 1/2/3 или микс». Дефолт — «выбрать рекомендованное и продолжить». Внеси правки в `design_spec.md` по **выбранному** направлению; невыбранные оставь файлами-прототипами. В Файле №4 зафиксируй: три ярлыка, выбранное, поверхность (Claude Design / Artifact / только spec — с причиной).

## Деградация (честность)
Если ни DesignSync, ни Artifact недоступны — отдай `design_spec.md` и текстовое описание экранов, явно пометив `PROTOTYPE: spec-only (<причина>)`. Не выдавай отсутствие прототипа за успех.
