# Гайд: визуальный прототип для утверждения

> Прототип нужен, чтобы пользователь **увидел** будущий интерфейс и утвердил его до написания книги. Две поверхности: основная — Claude Design (DesignSync), fallback — Artifact. В любом случае `design_spec.md` (Файл №4) пишется.

## Что собираем
Самодостаточные HTML-страницы (inline CSS на токенах дизайн-системы), показывающие:
1. **Дизайн-систему** — карточки компонентов (Button/Input/Card/StatusBadge/EmptyState/…), палитра, типографика.
2. **Ключевые экраны** из матрицы (хотя бы: каталог, деталь товара, создание заказа, деталь заказа покупателя с фотоконтролем, дашборд продавца, деталь заказа продавца с загрузкой фото) — с реалистичными данными и видимыми состояниями.

Стиль прототипа = токены из `design_system_spec.md` (тот же визуальный язык, что пойдёт в код).

## Путь A (основной): Claude Design через DesignSync

DesignSync синхронизирует локальную папку HTML-превью с design-system проектом на claude.ai/design; пользователь смотрит карточки в Design System pane и утверждает.

1. Собери локально папку `design/prototype/` с HTML-файлами. В первой строке каждого превью — маркер карточки:
   `<!-- @dsCard group="Components" -->` (или `group="Screens"`).
2. `DesignSync method=list_projects` → есть ли writable design-system проект. Нет — `create_project name="<Domain> UI"`.
3. `DesignSync method=finalize_plan projectId=<id> writes=["**/*.html"] localDir=<.../design/prototype>` → получи `planId`.
4. `DesignSync method=write_files planId=<planId>` (файлы через `localPath`).
5. Дай пользователю ссылку на проект в claude.ai/design; попроси посмотреть и утвердить/дополнить.

> Первый вызов может запросить design-доступ к claude.ai-логину (или `/design-login`). Если доступа нет / инструмент недоступен в сессии (headless/cron) — переходи на путь B и отметь это в Файле №4.

## Путь B (fallback): Artifact

1. Собери ОДНУ self-contained HTML-страницу (все CSS/JS inline, без внешних запросов — строгий CSP) с навигацией между экранами (якоря/таб-переключение) и карточками дизайн-системы. Перед этим загрузи навык `artifact-design`/`frontend-design` для калибровки.
2. Запиши файл и вызови `Artifact` с его путём, заголовком и favicon.
3. Дай пользователю ссылку на опубликованный Artifact; попроси утвердить/дополнить.

## Чекпоинт утверждения
Покажи: ссылку (Claude Design или Artifact) + список экранов + матрицу покрытия. Спроси, что поправить; дефолт — «утвердить и продолжить». Внеси правки пользователя в `design_spec.md`. В Файле №4 зафиксируй, какая поверхность использована (Claude Design / Artifact / только spec — с причиной).

## Деградация (честность)
Если ни DesignSync, ни Artifact недоступны — отдай `design_spec.md` и текстовое описание экранов, явно пометив `PROTOTYPE: spec-only (<причина>)`. Не выдавай отсутствие прототипа за успех.
