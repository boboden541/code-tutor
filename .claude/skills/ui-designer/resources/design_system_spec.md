# Дизайн-система — что фиксировать в Файле №4

> Дизайн-система — единый источник стиля. Описывается один раз в `design_spec.md`, реализуется через `<frontend_pack>/styling.md` (дефолт: токены + CSS Modules). Эстетику выводи из сути продукта (см. `screen_derivation.md` §0 и навык `frontend-design`), не из пресета.

## 1. Токены (значения выводятся из сути продукта)

| Группа | Токены | Примечание |
|--------|--------|-----------|
| Палитра | `--color-bg`, `--color-surface`, `--color-text`, `--color-muted`, `--color-accent`(+`-ink`), `--color-border`, `--color-success/-warning/-danger` | акцент — 1, выведенный из сути; статусные цвета — для StatusBadge |
| Типографика | `--font-sans`, `--font-display`, шкала `--fs-1..--fs-sm` | display-шрифт для заголовков задаёт характер |
| Spacing | `--space-1..--space-8` (4–32px) | единая шкала отступов |
| Форма | `--radius`, `--radius-sm`, `--shadow`, `--shadow-lg` | скругления/тени задают «настроение» |

Зафиксируй конкретные значения (hex, px, имена шрифтов), чтобы `book-author` положил их в `tokens.css` без догадок.

## 2. Компоненты (минимальный набор)

Перечисли компоненты с вариантами — это «алфавит», из которого собираются экраны:

- **Действия:** `Button` (варианты: primary / secondary / danger; состояния: default/disabled/pending).
- **Ввод:** `Input`, `Select`, `Textarea`, `FormField` (label + контрол + текст ошибки).
- **Данные:** `Card`, `StatusBadge` (цвет по статусу), `Price` (форматирование валюты), список/грид карточек.
- **Состояния:** `Spinner`, `EmptyState` (иконка + текст + CTA), `ErrorState` (текст + «Повторить»).
- **Каркас:** `Layout` (шапка+контент), `Nav` (роле-зависимые ссылки + вход/выход), `ProtectedRoute` (гейтинг по роли).

Для каждого — назови назначение и варианты; имена станут компонентами книги/проекта.

## 3. Тон и микротексты

Зафиксируй голос интерфейса (выводится из аудитории): заголовки, подписи кнопок, пустые состояния, тексты ошибок (человекочитаемые: 401→«Войдите снова», 403→«Недостаточно прав», 409→«Действие недоступно в текущем статусе»). Это убирает «сырые» технические сообщения.

## 4. Способ стилизации (фиксируется)

Дефолт — **токены (CSS-переменные) + CSS Modules**, без новых зависимостей. Запиши выбор в `design_spec.md` и в поле `styling` профиля. Если выбран Tailwind/UI-кит — отметь, что пак обязан его пинить и фронт-срез build-smoke gate должен пройти (иначе version-drift).

## 5. Связь с StatusBadge и доменом

Если в домене есть машина состояний (напр. заказ created→…→delivered), задай соответствие «статус → цвет токена + рус. подпись» здесь — чтобы UI единообразно показывал жизненный цикл.
