# Арт-дирекшн: три разных макета, сильная типографика, продающий UX

> Этот ресурс задаёт **планку визуального качества** шага `/tutor-design`. На выходе — **ровно три** прототипа-направления, каждое из которых вызывает эмоцию, продаёт продукт и адаптивно по умолчанию. Применяется поверх `design_system_spec.md` (токены) и `screen_spec_schema.md` (полнота экрана). Цель — чтобы пользователь выбирал из трёх **по-настоящему разных** идей, а не из трёх перекрашенных копий.

---

## A. Три направления — обязательно и они РАЗНЫЕ

`/tutor-design` собирает **три** визуальных направления. Они должны различаться **системно, а не цветом**. Каждое направление обязано отличаться от двух других минимум по **трём** осям из списка:

| Ось различия | Примеры полюсов |
|---|---|
| Типографический строй | гротеск+гротеск контрастной насыщенности · serif-display+гротеск-текст · моноширинный акцент · высококонтрастный «модный» serif |
| Композиция/сетка | асимметричный editorial · bento-сетка · split-screen (50/50) · слоистые перекрытия (overlap) · крупная вертикальная лента |
| Эмоция/позиционирование | премиум-бутик · дружелюбный соседский маркет · смелый журнал-редизайн · тёплый ремесленный |
| Цветовая стратегия | светлая воздушная · тёмная «премиум» · дуотон · насыщенный акцент по нейтрали |
| Характер формы | острые углы · мягкие крупные скругления · смешанная (карточки мягкие, кнопки-«пилюли») |
| Характер движения/hover | сдержанный (подсветка) · игривый (подскок/scale) · «дорогой» (плавный подъём тени) |

**Правило различности:** если три варианта отличаются только палитрой — это ОДНО направление, переделай. Дай каждому **ярлык** (1–2 слова) и **одно предложение позиционирования** (на кого и какую эмоцию). Каждое направление **самодостаточно** реализует ключевые экраны из матрицы покрытия (не «дизайн-система в вакууме»).

Пользователь выбирает одно направление (или просит микс) на чекпоинте — оно и фиксируется в `design_spec.md` (токены + шрифтовая пара + компоненты с состояниями). Два невыбранных остаются как файлы-прототипы.

## B. Сильная типографика (шрифтовые пары) — фундамент эмоции

Макет должен **вызывать эмоцию, а не просто передавать информацию** — а это в первую очередь типографика.

Правила для каждого направления:
1. **Пара шрифтов** с явными ролями и контрастом: *display* (заголовки — задаёт характер) + *text* (основной — читаемость). Контраст между ними реальный (по засечкам/насыщенности/ширине), не два почти одинаковых гротеска.
2. **Модульная шкала** с ощутимым контрастом размеров (напр. ×1.25–1.333): h1 крупный и выразительный, не «всё 16–20px». Заголовок-герой — это акцент страницы.
3. **Плотный leading и трекинг** у крупных заголовков; комфортный line-height (~1.5) у текста; ширина строки текста ≤ ~70 символов.
4. **Иерархия видна без цвета** — только размером/насыщенностью/регистром.

**Встроенный список проверенных пар** (Google Fonts; подключаются ссылкой `<link>` в прототипе и фиксируются в дизайн-системе). Выбирай по характеру продукта, по одной паре на направление — три РАЗНЫЕ пары:

| Характер | Display (заголовки) | Text (основной) |
|---|---|---|
| Премиум / редакционный | Fraunces, Playfair Display, Cormorant | Inter, Source Sans 3 |
| Современный продуктовый | Space Grotesk, Clash-подобный гротеск, Sora | Inter, IBM Plex Sans |
| Тёплый / ремесленный | Fraunces (soft), Bricolage Grotesque | Plus Jakarta Sans, Nunito Sans |
| Смелый / журнальный | Archivo Expanded, Anton, Syne | Manrope, Work Sans |
| Технологичный / точный | Space Grotesk + моно-акцент (JetBrains Mono) | IBM Plex Sans |

Если интернет/шрифт недоступны — используй системную пару (`ui-serif`/`Georgia` + `ui-sans-serif`/`system-ui`) и честно отметь это; принцип контраста и шкалы соблюдается всё равно.

## C. Продающий UX (UX/Marketing-driven)

Интерфейс должен **продавать**, а не только показывать данные. Там, где это уместно по сути продукта, включай:

1. **Социальное доказательство:** рейтинг (★ 4.9), число отзывов/заказов/клиентов, отзывы-цитаты с именем/аватаром, логотипы/«нам доверяют».
2. **Триггеры доверия:** продуктовая гарантия (для giftmarket — «фото готового подарка до доставки»), бейджи «Хит продаж» / «Выбор покупателей» / «Проверенный продавец», гарантия возврата/безопасной оплаты.
3. **Акцентный CTA:** на экране **один доминирующий** призыв к действию — визуально выделен (контраст, размер, «фирменная» заливка/градиент, тень в тон). Подпись — глагол-выгода («Подарить с гарантией», а не «Submit»). Вспомогательные действия — вторичны по весу.
4. **Состояния CTA проработаны полностью:** `default / hover / active / focus-visible / disabled / pending(загрузка)`. Hover ощутимый (сдвиг тени/подъём/подсветка/scale), фокус-кольцо для доступности, pending — спиннер+блокировка.

> **Честность, не тёмные паттерны.** Социальные доказательства и бейджи в прототипе — на **мок-данных**, помеченных как пример. Не имитируй дефицит/таймеры-обманки. Триггеры должны отражать реальную ценность продукта.

## D. Реалистичный контент — никакого Lorem Ipsum

Весь текст — осмысленный **микрокопирайтинг по сути продукта**:
- реальные названия товаров/разделов, человеческие описания, конкретные цены/города/даты;
- правдоподобные отзывы, имена, рейтинги;
- подписи кнопок и пустых состояний — на языке пользователя и выгоды.

**Запрещён** `Lorem ipsum`, «placeholder text», «текст-рыба», бессмысленные `aaa`/`123`. Если данных нет — придумай правдоподобные для домена (для giftmarket: «Букет пионов “Рассвет”», «Цветочный двор · Москва · ★4.9 (212 отзывов)»).

## E. Адаптивность по умолчанию

Каждый прототип адаптивен **сразу**, не «потом допилим»:
- `<meta name="viewport" content="width=device-width, initial-scale=1">`;
- относительные единицы и гибкие сетки (`flex`/`grid` с `auto-fit`/`minmax`), а не фиксированные ширины;
- медиа-запросы для мобильного: многоколоночное → одна колонка, увеличенные тач-цели (≥44px), нав сворачивается;
- **нет горизонтального скролла** у body; широкий контент (таблицы/ленты) скроллится внутри своего контейнера (`overflow-x:auto`);
- изображения/плейсхолдеры `max-width:100%`.

## F. КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО

Эти приёмы — признак устаревшего/непроработанного дизайна. Их наличие = прототип переделать:

1. **Устаревшие паттерны:**
   - скучная симметричная 12-колоночная сетка эпохи раннего Web 2.0 как единственный приём композиции;
   - плоские синие кнопки на белом фоне (generic-«bootstrap»);
   - карточки с жёсткими чёрными тенями (`box-shadow: 0 0 … #000` / `rgba(0,0,0,.5+)`). Тени — мягкие, полупрозрачные, в тон акценту/фону.
2. **Слепые зоны UX:**
   - кликабельные/интерактивные элементы без `:hover`-отклика;
   - элементы, которые выглядят кликабельными, но без `cursor:pointer`;
   - CTA без видимого `:focus-visible` (недоступность с клавиатуры).
3. **Контентные:** `Lorem ipsum` и любой текст-рыба.
4. **Псевдо-разнообразие:** три направления, отличающиеся только цветом.

## G. Связь с реализацией (чтобы это дошло до кода)

- Выбранное направление переносится в `design_spec.md`: токены (включая **шрифтовую пару** и ссылку на шрифты), компоненты **с описанными состояниями** (hover/focus/disabled/pending), триггеры доверия как переиспользуемые блоки (RatingStars, ReviewCard, TrustBadge).
- При стилизации «токены + CSS Modules» (дефолт) шрифтовая пара подключается через `<link>`/`@font-face`, зафиксированный в дизайн-системе; `:hover/:focus-visible/:disabled` — обязательная часть спецификации каждого интерактивного компонента, а не «на потом».
- `book-author` затем обязан реализовать эти состояния и триггеры (см. `book-author` §5c): «мёртвые» кнопки без hover/cursor и Lorem Ipsum — основание для FAIL валидатора.
