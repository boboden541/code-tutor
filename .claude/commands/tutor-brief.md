## Использование

```
/tutor-brief <описание проекта и стека>
```

**Параметр:** `<описание>` — свободный текст: какой FastAPI-проект вы хотите и с каким стеком.

## Пример

```
/tutor-brief Хочу сервис заметок с пользователями и тегами. Стек: FastAPI, PostgreSQL, SQLAlchemy, Alembic, JWT-аутентификация, Docker, GitLab CI.
```

**Ожидаемый результат:** файл `project_brief.md` (Файл №1) в корне проекта.

---

## Инструкция для LLM

### Этап 1: Загрузка роли
1. Прочитай `.claude/skills/project-brief/SKILL.md` — твоя персона и принципы.
2. Загрузи шаблон `.claude/skills/project-brief/examples/ideal_brief.md`.
3. Сверь термины с `.claude/skills/book-architect/resources/naming_conventions.md`.

### Этап 2: Деконструкция ввода
Из аргумента `<описание>` извлеки: цель продукта, стек, ключевые сущности, сценарии, границы MVP.

### Этап 3: Формирование Файла №1
1. Заполни `project_brief.md` строго по шаблону `ideal_brief.md`.
2. **Стек переноси дословно** из ввода пользователя — не подменяй дефолтным.
3. Поля, которые не выводятся из ввода, помечай `[уточнить]` — НЕ выдумывай.

### Этап 4: Сохранение
1. Если `project_brief.md` уже существует — спроси, перезаписать ли.
2. Сохрани `project_brief.md` в корне проекта.
3. Выведи: «Файл №1 создан: `project_brief.md`. Проверьте стек и поля `[уточнить]`. Следующий шаг: `/tutor-requirements project_brief.md`».
