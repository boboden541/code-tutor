## Использование

```
/tutor-requirements <путь к project_brief.md>
```

**Параметр:** путь к Файлу №1 (по умолчанию `project_brief.md`).

## Пример

```
/tutor-requirements project_brief.md
```

**Ожидаемый результат:** файл `requirements.md` (Файл №2) — опросник по 4 категориям с шаблонами и предложениями ИИ.

---

## Инструкция для LLM

### Этап 1: Загрузка роли
1. Прочитай `.claude/skills/requirements-interview/SKILL.md` — твоя персона и принципы.
2. Загрузи шаблон `.claude/skills/requirements-interview/examples/ideal_requirements.md`.
3. Сверь термины с `.claude/skills/book-architect/resources/naming_conventions.md`.

### Этап 2: Чтение Файла №1
Прочитай `project_brief.md`. Если файл не найден — сообщи и предложи запустить `/tutor-brief`.

### Этап 3: Генерация опросника
1. По каждой из 4 категорий (функциональные, пользовательские, интерфейсные, нефункциональные) сформируй вопросы.
2. Под каждым вопросом: **шаблон для заполнения** + **предложение ИИ** (разумный дефолт на основе Файла №1).
3. Опирайся на разделы Файла №1 (сценарии раздела 5, интерфейсы раздела 6) — углубляй их, а не дублируй.

### Этап 4: Сохранение и дружелюбный чекпоинт
1. Сохрани `requirements.md` рядом с Файлом №1.
2. Выведи: «Файл №2 создан: `requirements.md`. Заполните ответы под вопросами — **или сразу продолжайте с предложениями ИИ (дефолт)**. Следующий шаг: `/tutor-plan project_brief.md requirements.md`».
